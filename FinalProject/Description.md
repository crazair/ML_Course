# IncidentQA — автоматическая оценка и улучшение качества инцидентов в YouTrack с помощью LLM

---

## 1. Задача

**Название проекта:** *IncidentQA — автоматическая оценка и улучшение качества инцидентов в YouTrack с помощью LLM*

**Суть:** Система на базе LLM, которая анализирует тикеты-инциденты из YouTrack и:
- выставляет оценку качества оформления по набору критериев,
- даёт конкретные текстовые рекомендации по улучшению.

---

## 2. Данные

### Источники

- Экспорт реальных инцидентов из YouTrack (REST API или CSV-выгрузка).
- Разметка качества: вручную (или с помощью другого LLM-агента) размечаются 200–500 тикетов по критериям ниже — это станет ground truth для валидации.

### Критерии качества тикета (рубрика)

| Критерий | Описание |
|---|---|
| Заголовок | Конкретный, содержит суть проблемы |
| Шаги воспроизведения | Наличие и полнота |
| Ожидаемое/фактическое поведение | Чётко разделены |
| Окружение | Версия, платформа, среда |
| Приоритет | Обоснован и соответствует описанию |
| Полнота | Достаточно ли информации для диагностики |

Каждый критерий оценивается по шкале **0–2** (0 — отсутствует, 1 — частично, 2 — полностью). Итоговый балл — сумма.

### Объём

**300–500 тикетов** для валидации.

---

## 3. Постановка задачи и цели

### Кому полезно

- **Инженерам поддержки и разработчикам** — меньше времени на уточнение деталей, быстрее диагностика.
- **Тимлидам и менеджерам** — прозрачный контроль качества процесса заведения инцидентов.
- **Компании в целом** — сокращение MTTR (Mean Time To Resolve), меньше потерь от простоев.

### Измеримая польза

- Сокращение цикла уточнения инцидента на **20–40%** за счёт того, что репортер сразу получает фидбек.
- Снижение количества «мусорных» тикетов, требующих возврата на доработку.

## 4. Как решать?

**LLM:**

| Подход | Описание | Плюсы | Минусы |
|---|---|---|---|
| **Zero/Few-shot prompting** | GPT-4o / Claude через API с чётким промптом и рубрикой | Быстро, не нужны данные | Зависимость от внешнего API, стоимость |
| **Fine-tuning open-source модели** | Дообучение Mistral 7B / LLaMA 3 8B на размеченных тикетах | Контроль, нет зависимости от API | Нужно GPU, больше данных |
| **RAG + LLM** | LLM + база знаний с best practices (ITIL, внутренние стандарты) | Актуальные стандарты, объяснимость | Сложнее реализация |

## 5. Метрики

### «Человеческие» метрики (A/B и экспертная оценка)

- Доля тикетов, по которым репортер согласился с рекомендацией и улучшил описание.
- Субъективная оценка полезности рекомендаций по шкале 1–5 (опрос команды).
- Снижение количества тикетов со статусом «Требует уточнения».

> **Критерий успеха:** Снижение количества тикетов со статусом «Требует уточнения» в 2 раза.
